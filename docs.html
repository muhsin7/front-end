Using forms:

Include the events.js, and jquery in your html file.

<html lang="en" dir="ltr">
  <head>
    <script src="events.js" charset="utf-8"></script>
    <script src="jquery.js" charset="utf-8"></script>
  </head>
  <body>
    <!-- Form element is recognised with the help of class="form" -->
    <div class="form">
      <input type="text" name="" value="">
      <input type="text" name="" value="">
      <input type="text" name="" value="">
      <button type="submit" name="button"></button>
    </div>
  </body>
</html>

Forms

Form element attributes

  • action="<location>"
    • location: The location where the form would be processed/sent. This is like the classic action attribute in the vanilla form functionality
  • method="GET|POST|<any other types>"
    • GET: Usually used for basic requests, usually not advised for cisco project
    • POST: Usually used for larger and more requests, advised for cisco project
    • any other type: In case back end man specifies, use that
  • on-success="<JavaScript>"
    • JavaScript: When the form submission completes and the server acknowledges form submission, JavaScript would be executed as normal javascript. Just like the normal onclick attribute for buttons. You are given a variable inside JavaScript called response, this is the response by the server("successfully registered user","successfully updated post" etc.) which can be used like the following: 
      <div class="form" action="/auth/login" method="post" on-success="console.log(response)">
  <!-- stuff -->
</div>
      The above on-success attribute would log to the console the response to the console.
  • on-error="<JavaScript>"
    • JavaScript: Similar to on-success, however, this code would fire only when the server returns an error. Ex: "Invalid username and password". This time the variable you are given is called error. This can be used like the following 
      <div class="form" action="/auth/login" method="post" on-success="console.log(response)" on-error="console.log(error)">
  <!-- stuff -->
</div>
      The above on-error attribute would log to the console the error it received from the server(if there was an error in the first place).
  • on-success-reload="true|false"
    • true: The page would reload in case the request succeeds(like earlier)
  • on-success-redirect="<location>"
    • location: Would redirect the user to a specific page once the request succeeds
